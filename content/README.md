# 導入事例 記事追加ガイド

## 追加手順

1. `_template.md` をコピーして `{スラッグ}.md` という名前で保存する
2. フロントマターを埋める
3. 本文をペーストする
4. アイキャッチ画像を `public/images/cases/` に配置する
5. `npm run build` または dev サーバーを再起動すると自動で反映される

---

## ファイル命名規則

```
content/cases/{スラッグ}.md
```

- スラッグはURLになる（例: `clasian` → `/cases/clasian`）
- 英小文字・ハイフン区切りを推奨（例: `tanaka-foods`, `abc-logistics`）
- `_` で始まるファイルはスキップされる（`_template.md` はテンプレート用）

---

## フロントマター（必須7項目）

```yaml
---
slug: clasian
title: "記事タイトル"
client: "株式会社クラシアン"
industry: "住宅・設備"
employeeCount: "1,408名"
publishedAt: "2026-04-16"
thumbnail: "/images/cases/clasian.jpg"
---
```

| フィールド | 内容 |
|---|---|
| `slug` | URLパス。ファイル名と合わせる |
| `title` | 記事タイトル |
| `client` | 掲載企業名（正式名称） |
| `industry` | 業種 |
| `employeeCount` | 従業員数（表示文字列のまま） |
| `publishedAt` | 公開日（`YYYY-MM-DD` 形式） |
| `thumbnail` | アイキャッチ画像パス |

---

## 本文の書き方

本文は `---` の下からそのまま書く。Google Docs からのペーストも可。

```markdown
## 見出し（大）

段落は空行で区切る。

### 見出し（小）

> 担当者コメントや引用は `>` で始める。
> ページ上でハイライト表示される。
```

| 記法 | 表示 |
|---|---|
| `## テキスト` | 大見出し |
| `### テキスト` | 小見出し |
| `> テキスト` | 引用ブロック（ハイライト） |
| 空行区切り | 段落 |

---

## アイキャッチ画像

- 配置先: `public/images/cases/`
- ファイル名: スラッグに合わせる（例: `clasian.jpg`）
- 推奨サイズ: 横1200px 以上、16:9 または 3:2 比率
- フォーマット: `.jpg` / `.png` / `.webp`

画像がない場合はグレーの背景で表示される（ビルドエラーにはならない）。
